## A. 버그 리포트

### BUG-01: 결측치 포함 엑셀 업로드 시 페이지 오류

| 항목 | 내용 |
|------|------|
| **심각도** | 높음 |
| **SFR-03 관련** | 데이터 품질 관리 자동화 (결측치 탐지) |
| **재현 경로** | 기후기술수준조사 > 데이터 관리 > 엑셀 업로드 |
| **증상** | 빈 셀(NULL)이 포함된 엑셀 파일 업로드 시 "요청하신 페이지를 찾을 수 없습니다" 오류 발생 |
| **재현 조건** | 35열 중 1개 이상 값이 비어있는 상태로 업로드 |
| **임시 해결** | 결측 데이터 행을 하나씩 삭제하여 빈 셀 없는 상태로 재업로드 |
| **원인 추정** | 서버에서 NULL 값 파싱 시 예외 미처리 → 에러 페이지로 리다이렉트 |
| **기대 동작** | 업로드 전 빈 셀 검증 → 오류 메시지와 함께 해당 셀 위치 안내 |

### BUG-02: 데이터 0건/최신연도 부재 시 페이지 접근 불가 (일부 해결)

| 항목 | 내용 |
|------|------|
| **심각도** | 높음 |
| **SFR-03 관련** | 기존 대시보드 표출 오류 수정 및 안정화 |
| **재현 경로** | 기후기술수준조사 > 소분류별 데이터 > 전체 삭제 > 페이지 새로고침 |
| **증상** | 소분류 데이터를 전부 삭제한 뒤 해당 페이지 접근 시 "요청하신 페이지를 찾을 수 없습니다" 오류 |
| **근본 원인** | 메인 페이지가 2025년 데이터를 하드코딩으로 조회하도록 되어있었으며, 해당 연도 데이터가 없으면 페이지 렌더링 실패 |
| **1차 조치 (2026-04-06, 플랫폼 담당자)** | "입력된 제일 마지막 데이터를 가져오도록 수정" → 특정 연도 고정 → 최신 데이터 동적 조회로 변경 |
| **잔존 이슈** | 데이터 0건일 때의 빈 화면 처리는 아직 미확인 → SFR-03 "안정화" 범위에서 추가 점검 필요 |
| **기대 동작** | 데이터 0건이면 "등록된 데이터가 없습니다" 안내 표시 |

### BUG-03: 데이터 0건 상태에서 엑셀 업로드 실패

| 항목 | 내용 |
|------|------|
| **심각도** | 중간 |
| **SFR-03 관련** | 데이터 품질 관리 자동화 |
| **재현 경로** | BUG-02 상태에서 데이터 아카이브 > DB 엑셀파일 업로드 |
| **증상** | 데이터 0건 상태에서 엑셀 업로드가 동작하지 않음. 수기로 1건 입력한 후에야 엑셀 업로드 가능 |
| **재현 조건** | 데이터 전체 삭제 → 엑셀 업로드 시도 |
| **원인 추정** | 업로드 로직이 기존 데이터 존재를 전제로 UPDATE 방식으로 동작하며, INSERT(신규 등록) 경로가 없음 |
| **기대 동작** | 0건 상태에서도 엑셀 업로드로 신규 데이터 일괄 등록 가능 |

### BUG-04: 세부기술 탭 국가 선택 시 하드코딩 이슈 (기존)

| 항목 | 내용 |
|------|------|
| **심각도** | 중간 |
| **SFR-03 관련** | "국가 선택 시 핵심지표 카드·비교 차트 제목이 선택 국가에 따라 동적 변경되도록 수정 (현행 하드코딩 해소)" |
| **증상** | 세부기술 탭에서 국가를 변경해도 차트 제목·지표 카드가 기본 국가 기준으로 고정 표시 |
| **기대 동작** | 국가 선택 시 제목, 카드 값, 비교 차트 모두 동적 변경 |

### BUG-05: 전체 탭 세부기술 검색 시 페이지 오류

| 항목 | 내용 |
|------|------|
| **심각도** | 높음 |
| **SFR-03 관련** | "기존 대시보드(전체 탭/세부기술 탭) 표출 오류 수정 및 안정화" |
| **재현 경로** | 기후기술수준조사 > 전체 탭 > 변화 추이(전체 기술) > 상단 엑셀 다운로드 옆 (+) 클릭 > "세부기술" 선택 > 검색 |
| **증상** | "요청하신 페이지를 찾을 수 없습니다" 오류 |
| **재현 조건** | 국가비교 팝업에서 "세부기술" 옵션 선택 후 검색 실행 |
| **원인 추정** | 세부기술 검색 API가 미구현이거나, 세부기술 데이터 조회 쿼리에서 예외 발생 |
| **기대 동작** | 선택한 세부기술에 대한 국가별 비교 결과 표시 |

### BUG-06: 전체 탭 연도 필터 미작동

| 항목 | 내용 |
|------|------|
| **심각도** | 중간 |
| **SFR-03 관련** | "전체 탭 필터링·시각화 개선", "시계열 비교 기능 구현" |
| **재현 경로** | 기후기술수준조사 > 전체 탭 > 연도 선택 (2020~2025) |
| **증상** | 어떤 연도를 선택해도 동일한 화면 표시 (필터가 실제 데이터 조회에 반영되지 않음) |
| **재현 조건** | 2020, 2025 또는 다른 연도를 선택 |
| **원인 추정** | 연도 필터 UI는 존재하나 백엔드 쿼리에 연도 조건이 바인딩되지 않음 (하드코딩 또는 파라미터 누락) |
| **기대 동작** | 선택 연도의 데이터만 표시. 세부 필터(국가, 기술분야 등)에도 연도 조건 반영 |
| **비고** | SFR-03 시계열 비교 기능 구현의 전제 조건 — 연도 필터가 작동해야 2020↔2025 비교 가능 |

### 버그 공통 패턴 및 조치 요청

**유형 1 — 예외 처리 부재 (BUG-01~03, 05):**
데이터 비정상 상태(결측/0건/미지원 검색)에서 서버가 예외 처리 없이 404로 리다이렉트.

**유형 2 — 하드코딩/필터 미연동 (BUG-04, 06):**
UI에 필터/선택 기능은 있으나 실제 데이터 조회에 반영되지 않음.

SFR-03 "대시보드 안정화" 및 "시각화 개선" 범위에서 아래 조치 요청:
1. 업로드/삭제/조회 모든 경로에서 NULL, 0건, 중복, 미지원 검색에 대한 예외 처리 추가
2. 500 에러를 404로 리다이렉트하지 말고, 실제 에러 원인 메시지 표시
3. 연도/국가/기술분야 필터가 백엔드 쿼리에 실제 바인딩되는지 전수 점검
4. 데이터 전체 삭제 시 확인 다이얼로그 + 복구 안내

### REQ-01: 업로드 전 검증 미리보기 [우선순위: 상] [SFR-03 포함]

| 항목 | 내용 |
|------|------|
| **SFR-03 대응** | "데이터 갱신 시 자동 정합성 검증 로직 구현 (결측치 탐지, 합계 불일치 검출, 시계열 급변 탐지)" |
| **현재** | 엑셀 업로드 → 바로 DB 반영 → 오류 시 페이지 깨짐 (BUG-01 참조) |
| **요청** | 엑셀 업로드 → **검증 미리보기 화면** → 확인 후 DB 반영 |

**검증 항목 (발주처 제공 예정: 검증 룰 정의서):**

| 검증 | 내용 | 심각도 |
|------|------|:------:|
| 빈 셀 검출 | 행 번호 + 컬럼명 표시 | 오류 (업로드 차단) |
| 값 형식 오류 | 숫자 컬럼에 문자, 기술그룹에 선도/추격/후발 외 값 | 오류 |
| 값 범위 이탈 | 기술수준 0~100, 격차 0 이상 | 오류 |
| 중복 행 검출 | 동일 연도+소분류+국가 조합 중복 | 오류 |
| 기존 데이터 충돌 | 이미 등록된 연도와 겹치는 경우 | 경고 (덮어쓰기/건너뛰기 선택) |
| 시계열 급변 | 이전 연도 대비 기술수준 ±20%p 이상 변동 | 경고 (확인 후 진행) |
| 합계 불일치 | 세부기술 평균 ≠ 소분류 값 (오차 ±1 이내) | 경고 |

### REQ-02: 연도별 양식 템플릿 다운로드 [우선순위: 상] [추가 요청]

| 항목 | 내용 |
|------|------|
| **SFR-03 대응** | 직접 해당 없음 (DB 구조 고도화 시 양식 자동 생성 가능) |
| **현재** | 양식이 별도 공유되지 않아, 기존 데이터 다운로드 → 값 수정 → 재업로드 방식 |
| **요청** | 데이터 관리 화면에서 "빈 양식 다운로드" 버튼 |

**양식 종류:**
- 소분류별 데이터 양식 (현재 35열)
- 세부기술별 데이터 양식 (현재 38열)
- 연도 선택 시 해당 분류체계 기준 양식 자동 생성

### REQ-03: 소분류·세부기술 통합 업로드/다운로드 [우선순위: 상] [추가 요청]

| 항목 | 내용 |
|------|------|
| **SFR-03 대응** | 직접 해당 없음 (현행 분리 업로드 전제) |
| **현재** | 소분류별 / 세부기술별 **따로 다운로드, 따로 업로드** |
| **요청** | 연도 단위 **일괄 업로드** → 내부에서 자동 분리 저장 |

**현재의 문제점:**
- 같은 연도 데이터를 2번 나눠서 올려야 함 → 실수 유발, 시간 낭비
- 소분류와 세부기술 간 정합성을 수동으로 확인해야 함
- 하나만 올리고 다른 하나를 빠뜨리는 경우 발생

**기대 동작:**
- "2025년 수준조사 데이터 일괄 업로드" → 소분류 + 세부기술 한 번에 등록
- 또는 세부기술만 올리면 → 소분류는 자동 집계 생성 (REQ-01 검증에서 합계 확인)

### REQ-04: 분류체계 매핑 테이블 관리 [우선순위: 중] [SFR-03 포함]

| 항목 | 내용 |
|------|------|
| **SFR-03 대응** | "분류체계 간(38대↔100대↔45대) 매핑 관계를 시각적으로 탐색할 수 있는 네비게이션 UI 구현" |
| **현재** | 44대↔38대 매핑을 별도 엑셀로 관리, 시계열 비교 시 수동 대조 |
| **요청** | 플랫폼 내에서 분류체계 매핑 등록/수정 기능 |

**추가 요청 (SFR-03 기본 범위에 더해):**
- 매핑 유형(1:1, N:1, 1:N, 신규, 폐지) 시각 표시
- 매핑 등록/수정 권한을 관리자에게 부여 (현재는 코드 레벨)
- 매핑 변경 이력 관리

**매핑 등록 화면 예시:**
```
[2020(44대) 태양광] ──1:1──▶ [2025(38대) 태양광 기술]
[2020(44대) 수소제조] ──┐
                       ├─N:1─▶ [2025(38대) 수소 기술]
[2020(44대) 수소저장] ──┘
[2020(44대) 에너지 하베스팅] ──폐지──▶ (없음)
(없음) ──신규──▶ [2025(38대) 수열 기술]
```

### REQ-05: 연도 간 시계열 비교 뷰 [우선순위: 중] [SFR-03 포함]

| **현재** | 연도별 데이터를 각각 조회만 가능, 비교는 별도 작업 |
