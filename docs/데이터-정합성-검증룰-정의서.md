## 1. 검증 체계 개요

### 1.1 검증 시점

| 시점 | 설명 | 실행 방식 |
|------|------|-----------|
| **업로드 시** | 엑셀 파일 업로드 → DB 반영 전 사전 검증 | 자동 (미리보기 화면에서 결과 표시) |
| **저장 시** | 수기 입력/수정 → 저장 전 검증 | 자동 |
| **정기 점검** | 전체 데이터 정합성 일괄 점검 | 수동 (관리자 페이지에서 실행) |

### 1.2 판정 등급

| 등급 | 의미 | 동작 |
|:----:|------|------|
| **오류** | 데이터 무결성 위반. 반드시 수정 필요 | 업로드/저장 **차단**, 해당 셀 위치 표시 |
| **경고** | 이상값 가능성. 의도적일 수 있음 | 업로드/저장 **허용**, 확인 체크박스 후 진행 |
| **정보** | 참고 사항 | 리포트에만 표시, 동작 제한 없음 |

## 3. 수준조사 전용 검증 룰

### 3.1 소분류별 데이터 (35열)

**키 구성:** 제출연도 + 소분류명

| ID | 룰명 | 판정 | 조건 | 임계값 |
|:--:|------|:----:|------|:------:|
| S-01 | 기술수준 범위 | 오류 | 0 미만 또는 100 초과 | 0 ≤ x ≤ 100 |
| S-02 | 기술격차 범위 | 오류 | 음수 | x ≥ 0 |
| S-03 | 기술격차 상한 | 경고 | 15년 초과 | x ≤ 15 |
| S-04 | 최고국 100점 | 오류 | 5개국 중 100점인 국가가 없음 | max(5개국 수준) = 100 |
| S-05 | 최고국 격차 0 | 경고 | 100점 국가의 격차가 0이 아님 | 100점 국가 격차 = 0 |
| S-06 | 그룹 정합성 | 경고 | 100점 국가가 '선도'가 아님 | 100점 → 선도 |
| S-07 | 연구역량 범위 | 오류 | 0 미만 또는 100 초과 | 0 ≤ x ≤ 100 |
| S-08 | 상세기술수 범위 | 오류 | 0 이하 또는 30 초과 | 1 ≤ x ≤ 30 |
| S-09 | 소분류명 표준명 | 경고 | 등록된 분류체계의 표준명과 불일치 | 플랫폼 분류체계 DB 대조 |

### 3.2 세부기술별 데이터 (38열)

**키 구성:** 제출연도 + 소분류명 + 상세기술명

| ID | 룰명 | 판정 | 조건 | 임계값 |
|:--:|------|:----:|------|:------:|
| D-01 | 기술수준 범위 | 오류 | S-01과 동일 | 0 ≤ x ≤ 100 |
| D-02 | 기술격차 범위 | 오류 | S-02와 동일 | x ≥ 0 |
| D-03 | 기술격차 상한 | 경고 | S-03과 동일 | x ≤ 15 |
| D-04 | 최고국 100점 | 오류 | S-04와 동일 | |
| D-05 | 연구역량 범위 | 오류 | S-07과 동일 | 0 ≤ x ≤ 100 |
| D-06 | 중분류번호 연속 | 경고 | 중분류번호가 순차적이지 않음 | |
| D-07 | 상세기술번호 연속 | 경고 | 상세기술번호가 순차적이지 않음 | |

### 3.3 소분류↔세부기술 교차 검증

> 같은 연도의 소분류/세부기술 데이터가 모두 업로드된 경우 적용

| ID | 룰명 | 판정 | 조건 | 허용 오차 |
|:--:|------|:----:|------|:---------:|
| X-01 | 세부기술수 일치 | 오류 | 소분류의 상세기술수 ≠ 해당 소분류의 세부기술 행 수 | 정확 일치 |
| X-02 | 기술수준 정합 | 경고 | 세부기술 기술수준 평균 ≠ 소분류 기술수준 | ±1.0 |
| X-03 | 기술격차 정합 | 경고 | 세부기술 기술격차 평균 ≠ 소분류 기술격차 | ±0.5 |
| X-04 | 최고국 일치 | 정보 | 세부기술에서 산출한 최고국 ≠ 소분류 최고국 | — |
| X-05 | 소분류 누락 | 경고 | 세부기술은 있는데 대응되는 소분류가 없음 | — |
